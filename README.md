## ⚙️ Setup Steps (VS Code)

### Step 1 — Install Python & VS Code Extensions

Make sure you have:
- **Python 3.10+** installed → https://www.python.org/downloads/
- **VS Code** → https://code.visualstudio.com/

Install these VS Code extensions:
1. Open VS Code → Extensions (Ctrl+Shift+X)
2. Search and install **"Python"** (by Microsoft)
3. Search and install **"Live Server"** (by Ritwick Dey)

---

### Step 2 — Open the Project in VS Code

```
File → Open Folder → select the crop-ml-project/ folder
```

---

### Step 3 — Set Up Python Virtual Environment

Open the **VS Code Terminal** (Ctrl+` or View → Terminal) and run:

```bash
# Navigate to backend folder
cd backend

# Create virtual environment
python -m venv venv

# Activate virtual environment
# On Windows:
venv\Scripts\activate
# On Mac/Linux:
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

You should see `(venv)` in your terminal prompt when activated.

---

### Step 4 — Verify Data Files

Make sure these two files exist in `backend/data/`:
- `crop_dataset_clean.csv`
- `msp_dataset_clean.csv`

These are the cleaned CSV files generated by the ML pipeline scripts.  
If missing, run `crop_ml_framework.py` first and copy the outputs here.

---

### Step 5 — Start the Flask Backend

In the terminal (with venv activated, inside `backend/`):

```bash
python app.py
```

You should see:
```
[startup] Loading datasets …
[startup] Training ML models …
[startup] Ready ✓
 * Running on http://0.0.0.0:5000
```

**Keep this terminal open.** The API runs at `http://localhost:5000`

> 💡 First startup takes ~30–60 seconds to train the ML models.

---

### Step 6 — Open the Frontend with Live Server

1. In VS Code, open `frontend/index.html`
2. Right-click anywhere in the file
3. Select **"Open with Live Server"**
4. Your browser opens at `http://127.0.0.1:5500/frontend/index.html`

**The dashboard is now running!** 🎉

## 🔌 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/health` | Health check |
| GET | `/api/stats` | Dataset summary stats |
| GET | `/api/options` | Crops, states, model names |
| GET | `/api/eda/production` | Production trend data |
| GET | `/api/eda/area` | Area trend data |
| GET | `/api/eda/yield` | Yield trend data |
| GET | `/api/eda/heatmap?crop=wheat&top_n=12` | State heatmap data |
| GET | `/api/eda/top_states` | Top states per crop |
| GET | `/api/eda/yoy` | Year-on-year growth |
| POST | `/api/predict` | Predict production |
| GET | `/api/models/evaluation` | Model test metrics |
| GET | `/api/models/cv` | Cross-validation results |
| GET | `/api/models/importance?model=Random+Forest` | Feature importance |
| GET | `/api/models/residuals?model=Random+Forest` | Residual data |
| GET | `/api/forecast?crop=wheat&state=Rajasthan&steps=8&method=arima` | ARIMA/LSTM forecast |
| GET | `/api/msp/trend` | MSP trend data |
| GET | `/api/msp/district?crop=wheat` | District-wise MSP data |
| GET | `/api/msp/forecast?crop=wheat&steps=4` | MSP ARIMA forecast |

### Example Predict Request (POST /api/predict)

```json
{
  "crop":  "wheat",
  "state": "Rajasthan",
  "area":  3000000,
  "year":  2025,
  "yield": 3.5,
  "model": "Random Forest"
}
```

## 🛠️ Troubleshooting

**Problem: "Cannot reach the Flask API"**  
→ Make sure `python app.py` is running in the `backend/` folder with venv activated.

**Problem: CORS error in browser console**  
→ Flask-CORS is configured to allow all origins. Make sure you installed via `pip install -r requirements.txt`.

**Problem: `ModuleNotFoundError`**  
→ Make sure the venv is activated before running `python app.py`.

**Problem: Slow first load**  
→ Normal. Models train on startup (~30–60 seconds). Wait for "Ready ✓" in terminal.

**Problem: Live Server shows blank page**  
→ Right-click `frontend/index.html` specifically → "Open with Live Server".
